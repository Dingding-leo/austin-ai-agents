# ShipIt v1.1 — Iteration Notes

## Changes from v1 (based on self-critique)

### What Changed

1. **"Brief like a junior dev" removed** — Too generic, appears everywhere. Replaced with specific prompt structure that directly maps to each build step.

2. **Debug section reorganized** — Grouped by WHEN the error occurs (build time vs deployment time vs runtime). More actionable.

3. **Added "Before You Start" section** — Sets realistic expectations. Non-coders need to know what they're getting into before investing time.

4. **Simplified feature list** — v1 had 5 "stages" that could feel overwhelming. v1.1 condenses to 3 phases: PLAN → BUILD → SHIP.

5. **Added urgency/scarcity framing** — "20 free copies" framing removed (too pushy). Replaced with "First 20 beta testers get lifetime free updates."

## What Stays the Same

- Core problem (non-coders can't ship) is still real and validated
- Template structure is still solid
- Debug guides still address all 5 documented failure points
- Landing page approach is still correct
